Flexible-Origin Help Page


General Information

What is Flexible Origin?

Flexible Origin is a web server that can be controlled directly from the client using special request headers called x-fo headers.
It is the best way to reproduce specific responses without having to manipulate weird Apache or Nginx configurations.

Any use cases?

TONS. Your imagination is the limit.
X-FO Response Code

x-fo-statuscode-define

Goal Define the response status code
Header name x-fo-statuscode-define
Value syntax { 0,599 }
Example x-fo-statuscode-define: 200
X-FO Response Body

x-fo-body-showrequestheaders

Goal Show the request headers received by the origin in the body of the response in the format defined
Header name x-fo-body-showrequestheaders
Value syntax { json | text | xml }
Example x-fo-body-showrequestheaders: json

x-fo-body-showrequestbody

Goal Show the request body received by the origin in the body of the response in the format defined
Header name x-fo-body-showrequestbody
Value syntax { plain }
Example x-fo-body-showrequestbody: plain

x-fo-body-info

Goal Send relevant information through the response body (request headers received by the origin, origin IP, origin hostname, request method, URL)
Header name x-fo-body-info
Value syntax { json | text | xml }
Example x-fo-body-info: json

x-fo-body-compressed

Goal Return a compressed object in the required format
Header name x-fo-body-compressed
Value syntax { zip | gzip | bzip | deflate | br }
Example x-fo-body-compressed: gzip

x-fo-body-binary

Goal Return a binary random object with the selected size in bytes
Header name x-fo-body-binary
Value syntax { Number_of_bytes }
Example x-fo-body-binary: 20

x-fo-body-text

Goal Return a text based object of the desired format
Header name x-fo-body-text
Value syntax { css | html | js | plain | indexhtml }
Example x-fo-body-text: css

x-fo-body-path

Goal When "true" the Flexible Origin will respect the incoming path to look the asset
Header name x-fo-body-path
Value syntax { true | false }
Example x-fo-body-path: true
X-FO Response Headers

x-fo-header-add

Goal Add header in the response from origin. You must define a different number for each header you are trying to set
Header name x-fo-header-add-{number}
Value syntax { header_name=header_value }
Example x-fo-header-add-1: Cache-Control=max-age=0s

x-fo-header-remove

Goal Remove header coming from the origin in the response
Header name x-fo-header-remove
Value syntax { header_name }
Example x-fo-header-add-1: Content-Type

x-fo-header-addcookie

Goal Add cookie in the response from origin. You must define a different number for each cookie you are trying to set
Header name x-fo-header-addcookie-{number}
Value syntax { header_name=header_value }
Example x-fo-header-modify-1: thisIsAcookie=IamACookie

x-fo-header-modify

Goal Modify a reponse header coming from the origin. You must define a different number for each cookie you are trying to set
Header name x-fo-header-modify-{number}
Value syntax { cookie_value }
Example x-fo-header-addcookie-1: Cache-Control=max-age=0s

x-fo-header-basicauth

Goal Enable basic authentication
Header name x-fo-header-basicauth
Value syntax { user=yourUserName; pass=yourPassword }
Example x-fo-header-basicauth: user=xap; pass=thisisasecret
X-FO Action Headers

x-fo-action-delay

Goal Make the Flexible Origin delay its response by the amount of seconds you need
Header name x-fo-action-delay
Value syntax { time_in_seconds }
Example x-fo-action-delay: 15
FO API

Push asset

Goal This API allows the user to upload files to the Flexible Origin
Endpoint /pushasset
Method POST
Parameters Any of:
Query String - path
Form post - path
Example POST www.yourflexibleorigin.com/pushasset?path=your/path and file in the body

Download Origin Logs

Goal This API allow the user to download the log file of the flexible origin
Endpoint /getlog
Method GET
Parameters Query String - format
Where "format" can be "plain" or "download" file
Example GET www.yourflexibleorigin.com/getlog?format=plain
